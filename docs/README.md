# PSKC Site Editing Guide

This site is an Astro static site. Most editing work falls into one of three
paths:

| Work type | Edit here | Output URL |
| --- | --- | --- |
| Full pages with Astro components | `src/pages/*.astro` | Based on the file name |
| Markdown-generated news posts | `src/content/news/*.md` | `/<filename>/` |
| Navigation links and shared site data | `src/data/site.ts` | Header and shared components |

Do not edit `dist/` directly. It is generated by `npm run build`.

## Adding New Astro Pages

Use an Astro page when the page needs custom layout, components, sections, cards,
or other structured markup.

1. Create a file in `src/pages/`.
2. Name the file after the route you want.
3. Import `BaseLayout` and any shared components you need.
4. Add the page to `navItems` in `src/data/site.ts` if it should appear in the
   header navigation.
5. Run `npm run build` before publishing.

Examples:

| File | URL |
| --- | --- |
| `src/pages/about.astro` | `/about/` |
| `src/pages/resources.astro` | `/resources/` |
| `src/pages/volunteer.astro` | `/volunteer/` |

Minimal page:

```astro
---
import BaseLayout from "../layouts/BaseLayout.astro";
import Hero from "../components/Hero.astro";
import Section from "../components/Section.astro";
---

<BaseLayout
  title="Volunteer"
  description="Volunteer opportunities with the Psychedelic Society of Kansas City."
>
  <Hero
    compact
    eyebrow="Volunteer"
    titleLines={["Help", "PSKC."]}
    intro="Support community education, events, and harm reduction work in Kansas City."
    image="/assets/images/PSKC%20FB_BANNER-01.jpeg"
  />

  <Section>
    <article class="prose-site">
      <h1>Volunteer</h1>
      <p>Add page content here.</p>
    </article>
  </Section>
</BaseLayout>
```

Astro routes use the `base` setting from `astro.config.mjs` during the build, so
write internal page links as site-root paths such as `/about/`, not
`/pskc-staging/about/`, inside Astro components.

When linking assets from Astro files, pass root-relative paths through helpers or
components that already call `withBase()`. For example, the `Hero` and `Button`
components handle this for their `image` and `href` props.

## Adding Markdown-Generated Pages

Published Markdown pages currently use the `news` content collection:

```text
src/content/news/
```

Each Markdown file is rendered by `src/pages/[slug].astro`. The filename becomes
the route slug.

Example:

```text
src/content/news/city-council-statement.md
```

Builds to:

```text
/city-council-statement/
```

The post is also listed on `/news/`, because `src/pages/news.astro` loads every
entry from the `news` collection and sorts them by date.

Use this frontmatter shape:

```md
---
title: "Post title"
date: 2026-05-01
description: "Short summary shown on the News page and used for metadata."
categories:
  - News
tags:
  - Kansas City
---
```

Required fields:

- `title`
- `date`
- `description`

Optional fields:

- `categories`
- `tags`

The schema lives in `src/content.config.ts`. If you add new frontmatter fields
that should be validated or displayed, update that schema and the rendering code
in `src/pages/[slug].astro`.

Use `.md` files for normal posts. The collection pattern also matches `.mdx`,
but this repository does not currently include the `@astrojs/mdx` integration.
Do not add MDX-only syntax or component imports unless that integration is added
to `package.json` and `astro.config.mjs`.

## Updating The Navigation Bar

The header navigation is controlled by `navItems` in `src/data/site.ts`.

```ts
export const navItems = [
  { title: "News", href: "/news/" },
  { title: "Events", href: "/events/" },
  { title: "Resources", href: "/resources/" },
];
```

To add a link:

1. Make sure the page exists.
2. Add a `{ title, href }` item to `navItems`.
3. Use a trailing slash for internal routes, such as `/volunteer/`.
4. Do not include `/pskc-staging` in `href`; `Header.astro` calls `withBase()`
   during rendering.

The order of `navItems` is the visible order in both the desktop nav and mobile
menu. Keep the list short enough to fit the header on desktop. If a page is
important but does not fit in the main nav, link it from a page body, card, or
footer action instead.

External links can also be added to `navItems`, but the current pattern keeps
the main header focused on site pages and uses buttons or social links for
external destinations.

## Markdown Formatting Reference

Markdown posts render inside:

```astro
<article class="site-container prose-site">
```

Most visual formatting for Markdown output is in `src/styles/global.css` under
the `.prose-site` selectors. If a new Markdown pattern needs custom styling, add
the style there.

Common formatting:

````md
# Main heading

## Section heading

### Smaller heading

Paragraph text with **bold**, *italic*, and ~~strikethrough~~ text.

[Link text](https://example.com)

- Bullet item
- Bullet item

1. Numbered item
2. Numbered item

> Pull quote or quoted statement.
>
> <cite>Attribution</cite>

`inline code`

```text
code block
```
````

Astro has GitHub-flavored Markdown enabled by default in this project version,
so tables, task lists, strikethrough, autolinks, and footnotes are available.

Table:

```md
| Session | Topic | Format |
| --- | --- | --- |
| April | Harm reduction | Talk and Q&A |
| May | Integration | Panel |
```

Task list:

```md
- [x] Publish recap
- [ ] Add next event date
```

Footnote:

```md
Community education remains central to PSKC's work.[^note]

[^note]: Add source notes or short clarifications here.
```

Images in Markdown should include the GitHub Pages base path because Markdown
image URLs are rendered directly into the page:

```md
![Alt text](/pskc-staging/assets/images/news/example-photo.jpg)
```

Place post images under:

```text
public/assets/images/news/
```

Raw HTML is allowed in Markdown and is useful for special blocks that already
have CSS. The existing `fact-box` class is one example:

```html
<div class="fact-box">
<p><strong>Key Facts at a Glance</strong></p>
<ul>
<li><strong>Status:</strong> Example note</li>
<li><strong>Focus:</strong> Public education</li>
</ul>
</div>
```

Use raw HTML sparingly. Prefer normal Markdown for headings, lists, links,
quotes, tables, and images. Add a custom class only when the content needs a
distinct visual treatment.

## Before Publishing

Run a build:

```sh
npm run build
```

Or through the Nix shell:

```sh
nix develop -c npm run build
```

Pushing to `main` runs `.github/workflows/astro.yml`, builds the site, and
deploys the generated `dist/` artifact to GitHub Pages.
